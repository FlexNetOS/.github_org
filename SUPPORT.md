# Getting help

This document describes where to ask for help across every
[@FlexNetOS](https://github.com/FlexNetOS) repository that inherits it
(i.e. has no local `SUPPORT.md`). A repo-local copy always overrides.

## Decision tree

| What you need | Where to go |
| --- | --- |
| Report something demonstrably broken (crash, wrong output, regression). | Open an issue using the **Bug report** template on the affected repo. |
| Propose a new feature or a meaningful behavior change. | Open an issue using the **Feature request** template on the affected repo. |
| Report missing, wrong, or confusing documentation. | Open an issue using the **Documentation** template on the affected repo. |
| Ask a how-do-I, design, or architecture question. | Use [GitHub Discussions](../../discussions) on the affected repo. |
| Report a security problem. | Do **not** open a public issue — follow [SECURITY.md](SECURITY.md). |
| Raise a Code of Conduct concern. | Email <revenaugh.david@gmail.com> per [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). |

## Search before posting

Most questions have already been asked. Before opening anything new:

1. Search the repo's [Issues](../../issues?q=) — both open and closed.
2. Search the repo's [Discussions](../../discussions?discussions_q=).
3. Search the repo's README and `docs/` for the keyword.

## Response expectations

These are open-source projects maintained at the discretion of their
contributors. There is no commercial SLA. Best effort:

| Channel | Best-effort response |
| --- | --- |
| Security reports | acknowledged within 48 hours — see [SECURITY.md](SECURITY.md) |
| Bug reports with a minimal reproduction | triaged within ~1 week |
| Bug reports without a reproduction | may be closed and re-opened once a reproduction lands |
| Feature requests | discussed in the issue; no commitment to ship |
| Discussions / questions | best effort; contributors are encouraged to answer each other |

## Commercial support

There is no paid support tier for FlexNetOS projects at this time.
